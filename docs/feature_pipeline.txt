
FEATURE PIPELINE --> Convierte los datos de la fuente de atos en Features y los almacena para pasarlos al entrenamiento
y predicciones.

1. Extrae los datos desde una fuente (por ejemplo parquet, bbdd, APIs...)
2. Transformar esos datos en features para entrenar o predecir (feature engineering, limpieza
agregaciones, lags...).
3. Guardar estos datos ya transfromados (features) en un lugar accesible para entrenamiento y predicción 
(FEATURE STORE).


FEATURE STORE -->
1. Es un sistema centralizado donde puedes gurarda y versionar features ya procesados.
2. Reutilizar esos mismos features en entrenamiento y predicción sin duplicar lógica.
3. Servir features en batch o en tiempo real. 
4. Mantener consistencia entre entrenamiento y servin --> uno de los mayores problemas en producción.
